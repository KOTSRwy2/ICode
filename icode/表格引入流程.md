# 表格组件引入流程规范 (PyQt-Fluent-Widgets)

在项目中引入并显示数据表格（如 fMRI 激活簇表、连接矩阵数据等）时，为了确保表格样式不被破坏并与 `PyQt-Fluent-Widgets` 的整体主题（深浅色模式）完美适配，请严格遵循以下规范流程。

## 1. 使用 PyQt-Fluent-Widgets 原生表格组件

在 PyQt 端直接渲染表格数据时，强烈建议使用 `qfluentwidgets` 提供的 `TableWidget`。

### 1.1 导入组件
```python
from qfluentwidgets import TableWidget
from PyQt5.QtWidgets import QTableWidgetItem
```

### 1.2 初始化与配置
```python
table = TableWidget(self)

# 设置行列数
table.setRowCount(10)
table.setColumnCount(5)

# 设置表头
table.setHorizontalHeaderLabels(['ID', '脑区', '激活强度', '体素数', 'p-value'])

# 自动调整列宽
table.horizontalHeader().setStretchLastSection(True)

# 填充数据
for i in range(10):
    table.setItem(i, 0, QTableWidgetItem(str(i+1)))
    table.setItem(i, 1, QTableWidgetItem(f"ROI_{i}"))
    # ...
```
**优点**：原生 `TableWidget` 内置了深浅色主题适配，无缝对接 `ThemeManager`。

## 2. 在 WebEngineView 中渲染 HTML 表格

如果表格是通过第三方库（如 `nilearn.reporting.get_clusters_table().to_html()` 或 `pandas.DataFrame.to_html()`）生成并在 `QWebEngineView` 中显示的，必须注意 **CSS 样式隔离**。

### 2.1 避免全局 CSS 污染（核心修复点）
过去在注入主题 CSS 时，使用了激进的通配符：
```css
/* 错误示例：会使得表格边框、阴影全部消失，导致表格无法显示 */
* { border: none !important; box-shadow: none !important; }
```
**必须修改为精准作用域的 CSS：**
```css
/* 正确示例：只针对 body 和基础容器做背景透明处理 */
body, html { 
    background-color: transparent !important; 
    margin: 0; 
    padding: 0; 
}
```

### 2.2 注入适配 Fluent 主题的表格样式
为了让 HTML 表格看起来像 Fluent UI 的控件，可以在生成的 HTML 中注入专门针对 `table` 的样式：

```python
css_table_style = """
<style>
    /* 适配 Fluent 主题的表格样式 */
    table {
        width: 100%;
        border-collapse: collapse;
        font-family: "Segoe UI", "Microsoft YaHei", sans-serif;
        color: inherit; /* 继承外层文字颜色，自动适配深浅色 */
    }
    th, td {
        padding: 8px 12px;
        border-bottom: 1px solid rgba(128, 128, 128, 0.2);
        text-align: left;
    }
    th {
        font-weight: 600;
        background-color: rgba(128, 128, 128, 0.1);
    }
    tr:hover {
        background-color: rgba(128, 128, 128, 0.05);
    }
</style>
"""

if '<head>' in html_content:
    html_content = html_content.replace('<head>', f'<head>{css_table_style}')
```

## 3. 弹窗展示表格数据逻辑

如果表格内容过多，不适合在主卡片中直接展示，可以将其放入弹窗中。应使用 `qfluentwidgets` 提供的 `MessageBox` 或自定义弹窗容器。

### 3.1 弹窗实现示例
```python
from qfluentwidgets import MessageBoxBase, SubtitleLabel
from PyQt5.QtWidgets import QVBoxLayout

class TableMessageBox(MessageBoxBase):
    def __init__(self, parent=None):
        super().__init__(parent)
        self.titleLabel = SubtitleLabel('详细数据表')
        
        # 将上面定义的 TableWidget 加入弹窗
        self.table = TableWidget()
        # ... 初始化 table 数据 ...
        
        self.viewLayout.addWidget(self.titleLabel)
        self.viewLayout.addWidget(self.table)
        
        # 设置弹窗最小尺寸
        self.widget.setMinimumWidth(600)
        self.widget.setMinimumHeight(400)

# 调用
dialog = TableMessageBox(self.window())
dialog.exec_()
```

## 4. 总结

1. **优先原生**：尽量使用 `qfluentwidgets.TableWidget`，它自带主题切换功能。
2. **谨慎写 CSS**：在使用 `QWebEngineView` 时，坚决杜绝 `* { border: none; }` 这种毁灭性全局样式，避免破坏第三方库自动生成的表格。
3. **响应式色彩**：在 HTML 中使用 `color: inherit` 和 `rgba` 透明度背景，使得表格能在浅色和深色模式下自适应。
